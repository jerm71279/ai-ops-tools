Migration Steps

Phase 1: Preparation

  1. Export MikroTik config backup
  2. Document all current DHCP leases
  3. Update UDM-Pro firmware
  4. Pre-configure all networks in UniFi (can do while MikroTik is live)

Phase 2: Network Creation (Do in UniFi before cutover)

  1. Create all 5 infrastructure networks
  2. Create all 84 unit networks
  3. Configure DHCP settings for each
  4. Create firewall rules
  5. Create port forwarding rules
  6. Add static DHCP reservations

Phase 3: Cutover

  1. Schedule maintenance window
  2. Disconnect MikroTik WAN
  3. Connect UDM-Pro WAN (Port 9 SFP+ or Port 11 RJ45)
  4. Configure WAN static IP
  5. Connect trunk port to switch
  6. Verify connectivity per-VLAN
  7. Test stamp machine port forwards
  8. Monitor for issues

Phase 4: Validation

Important Considerations

UDM-Pro Capabilities

  • Max Networks: 100+ networks supported (89 is within capacity)
  • Performance: 3.5 Gbps IDS/IPS, 10 Gbps routing
  • Memory: 4GB RAM - handles complex configs well
  • Bonus: Built-in UniFi Protect NVR slot (optional)

Alternative: Keep VLANs on Existing Switch

If UDM-Pro struggles with 89 VLANs, consider: 1. Use UDM-Pro for routing/firewall only 2. Keep VLAN configuration on managed switch 3. UDM-Pro handles WAN, NAT, DHCP for core networks only

Recommendations

  1. Test with subset first - Create 10-15 VLANs, verify stability
  2. Monitor memory/CPU - Check after adding all networks
  3. Consider UniFi Enterprise - If performance issues, UXG-Pro handles more
  4. Batch creation - Use UniFi API to create networks faster

UniFi API Bulk Network Creation (Optional)

For faster setup, use the UniFi API. Python script available upon request to automate network creation.

Rollback Plan

If migration fails: 1. Disconnect UDM-Pro 2. Reconnect MikroTik 3. Verify all services restored 4. Document issues encountered 5. Plan remediation
