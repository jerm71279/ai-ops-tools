Security Findings

Critical Router Admin Interface Exposed

ASUS WRT admin interface accessible on port 80 at both locations. Consumer-grade routers lack enterprise security features.

Critical SMB Services on Multiple Hosts

Windows file sharing active on workstations. SMB signing "enabled but not required" - potential for relay attacks.

Moderate Unencrypted VoIP SIP

SIP traffic on TCP/5060 may be unencrypted. SIP TLS (5061) is available on Yealink devices but not configured.

Moderate Multiple HTTP Admin Interfaces

Printers, switches, and VoIP devices have web interfaces accessible over HTTP. Verify strong passwords on all devices.

Positive Location #2 Mini PC Firewall

NucBox_K8 has all ports filtered - good security posture. Firewall properly configured on this device.

Recommendations

1

Replace ASUS routers with SonicWall TZ

Consumer-grade routers lack enterprise security features. Deploy SonicWall TZ at both locations for proper firewall, VPN, and threat protection.

2

Enable SMB Signing and Disable SMBv1

Configure Windows hosts to require SMB signing. Disable legacy SMBv1 protocol to prevent exploitation.

3

Implement Network Segmentation (VLANs)

Create separate VLANs for Data, Voice, Guest, IoT/Printers, and Security systems. Requires managed switch configuration.

4

Configure SIP over TLS for VoIP

Enable encrypted SIP (port 5061) on Yealink devices to protect voice communications.

5

Deploy Network Monitoring Solution

Implement SNMP monitoring, alerts, and logging. Add devices to NinjaOne/RMM for centralized management.

6

Implement 802.1X Authentication

Long-term goal: Deploy network access control to authenticate devices before allowing network access.
