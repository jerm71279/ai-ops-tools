PHASE 1
Network Discovery & Assessment ‚Üí Onsite Pre-sales
Initial Site Survey
1. Schedule On-site Discovery Session
  • Contact customer to schedule discovery session
  • Prepare site survey checklist and tools
2. Document Current Network Topology
SonicWall MikroTik UniFi
  • Identify existing network devices:
    • SonicWall firewalls (model, firmware, licenses)
    • MikroTik routers (model, RouterOS version)
    • UniFi gateways, switches, APs (model numbers)
    • Note serial numbers for warranty/support
  • Map IP addressing scheme and VLAN structure:
    • Current IP addressing scheme
    • VLAN IDs and purposes
    • DHCP scopes and ranges
    • Static IP assignments
  • Document internet connection type and bandwidth:
    • ISP provider and circuit type
    • Bandwidth (upload/download)
    • Static IPs available
    • Gateway/DNS information
  • Identify critical systems and devices:
    • Servers and their network requirements
    • VoIP systems
    • Security systems (cameras, access control)
    • IoT devices
3. Assess Physical Infrastructure
UniFi
  • Evaluate existing cabling (Cat5e/6, fiber):
    • Test existing Cat5e/6 runs
    • Identify fiber connections
    • Note cable lengths and patch panel organization
    • Document any cable management needs
  • Check rack space availability:
    • Available rack units (RU)
    • Rack dimensions and depth
    • Cable management space
    • Ventilation and cooling
  • Identify mounting locations for access points:
    • Ceiling height and material
    • Wall mount feasibility
    • Coverage requirements per area
    • Aesthetic considerations
  • Verify power availability (PoE requirements, UPS):
    • PoE switch capacity needed
    • PoE+ for UniFi APs (802.3at/bt)
    • UPS capacity and runtime
    • Power outlet availability
4. Gather Business Requirements
SonicWall UniFi
  • Number of users and devices:
    • Employees per department
    • Workstations, laptops, mobile devices
    • Printers and peripherals
    • Growth projections (1-3 years)
  • Bandwidth requirements per department/function:
    • Per-user bandwidth needs
    • Department-specific requirements
    • Application bandwidth (VoIP, video, cloud)
    • Peak usage times
  • Guest network requirements:
    • Guest access method (portal, vouchers, password)
    • UniFi guest portal customization needs
    • Bandwidth limitations
    • Access duration/time limits
  • Security and compliance requirements:
    • Industry compliance (HIPAA, PCI-DSS, etc.)
    • SonicWall security services needed (IPS, GAV, CF)
    • Content filtering requirements
    • VPN access requirements
Network Analysis
5. Conduct Wireless Site Survey (if applicable)
UniFi
  • Identify RF interference sources:
    • Use UniFi WiFiman app for site survey
    • Identify 2.4GHz/5GHz/6GHz interference
    • Check for neighboring APs
    • Note RF-blocking materials
  • Determine AP placement and quantity:
    • Coverage area per AP type (U6/U7 Lite/LR/Pro)
    • High-density area requirements
    • Outdoor AP needs
    • Channel planning for minimal interference
  • Plan channel allocation:
    • 2.4GHz channel selection (1, 6, 11)
    • 5GHz channel width (20/40/80/160MHz)
    • DFS channels consideration
    • Channel bonding strategy
6. Analyze Traffic Patterns and Bottlenecks
MikroTik UniFi
  • Current bandwidth utilization
  • Top talkers and applications
  • Identify bottlenecks
  • QoS requirements for MikroTik/UniFi
7. Document Security Gaps and Vulnerabilities
SonicWall
  • Audit current SonicWall firewall rules
  • Review existing security policies
  • Identify vulnerabilities
  • Document security incidents
8. Identify Single Points of Failure
SonicWall MikroTik UniFi
  • Single points of failure
  • Backup internet connection needs
  • Redundant gateway requirements
  • Switch stacking/failover options
9. Review Backup and Redundancy Needs
SonicWall MikroTik UniFi
  • Configuration backup procedures
  • Failover mechanisms (SonicWall HA, MikroTik VRRP)
  • UniFi controller backup strategy
Documentation Deliverables
10. Create Network Diagram (Current State)
SonicWall MikroTik UniFi
  • Current state topology
  • All SonicWall, MikroTik, UniFi devices
  • VLAN layout
  • IP addressing scheme
  • Use draw.io, Visio, or Lucidchart
11. Document Findings Report
  • Site survey summary
  • Current state assessment
  • Identified issues and risks
  • Recommendations overview
PHASE 2
Recommendation & Design ‚Üí After Phase 1 before SOW Quote
Device Recommendations
12. Core Router/Firewall Selection
SonicWall MikroTik UniFi
  • SonicWall Firewall Options:
    • TZ series (small office): TZ270, TZ370, TZ470, TZ570
    • NSa series (mid-size): NSa 2700, NSa 3700, NSa 4700
    • NSsp series (large/enterprise): NSsp 10700, NSsp 11700
    • Best for: Advanced security features, IPS, content filtering
  • MikroTik Router Options:
    • Small office: hEX S, RB5009
    • Mid-size: CCR1009, CCR1036, CCR2004
    • Enterprise: CCR2116, CCR2216
    • Consider RouterOS license level and throughput needs
    • Best for: Advanced routing, MPLS, BGP requirements
  • UniFi Gateway Options:
    • Small office: Dream Machine (UDM), Dream Machine SE
    • Mid-size: Dream Machine Pro (UDM-Pro)
    • Enterprise: UXG-Pro, UXG-Max
    • Consider IPS/IDS throughput and DPI requirements
    • Best for: Unified management, simplified deployments
  • Selection criteria:
    • Throughput requirements and user count
    • Feature needs (VPN, IPS, content filtering)
    • Management preferences (cloud vs local)
    • Budget and licensing considerations
  • Throughput requirements:
    • Firewall throughput (Mbps)
    • IPS/GAV throughput
    • VPN throughput
    • SSL inspection throughput
    • Concurrent connections needed
Design Documentation
13. Create Proposed Network Topology Diagram
SonicWall MikroTik UniFi
  • Include all recommended SonicWall, MikroTik, UniFi devices
  • Show VLAN segmentation
  • Indicate redundancy paths
  • Label all connections and ports
14. Define IP Addressing and VLAN Scheme
SonicWall MikroTik UniFi
  • Management VLAN:
    • Dedicated VLAN for network equipment
    • Recommended: VLAN 10, 10.x.10.0/24
    • Access restricted to IT staff
    • SonicWall, MikroTik, UniFi management IPs
  • User/Data VLANs:
    • Department-based VLANs
    • Example: VLAN 20 (10.x.20.0/24) - Staff
    • Appropriate firewall rules between VLANs
    • Configure in UniFi Networks
  • Voice VLAN:
    • Dedicated VLAN for VoIP phones
    • QoS priority on UniFi switches
    • Recommended: VLAN 30, 10.x.30.0/24
    • DSCP tagging for voice traffic
  • Guest VLAN:
    • Isolated network for guests
    • Internet-only access (no local network)
    • Recommended: VLAN 40, 10.x.40.0/24
    • UniFi guest portal integration
    • Bandwidth limitations
  • IoT/Security VLAN:
    • Isolated VLAN for IoT devices
    • Security cameras, access control
    • Recommended: VLAN 50, 10.x.50.0/24
    • Restricted internet access via SonicWall
15. Plan Wireless Network Architecture
UniFi
  • SSID naming and security:
    • Corporate SSID (WPA3-Enterprise or WPA2/3)
    • Guest SSID with captive portal
    • IoT SSID (separate from corporate)
    • Hidden SSIDs if required
  • AP coverage map:
    • AP placement on floor plan
    • Coverage overlap for roaming
    • Minimum -67dBm signal strength
    • High-density areas (conference rooms)
16. Design Security Policies
SonicWall MikroTik UniFi
  • Firewall rules:
    • Zone-based security policies
    • Allow/deny rules between VLANs
    • Application control rules
    • Geo-IP filtering
    • Schedule-based rules
  • Access control lists:
    • MikroTik firewall filter rules
    • UniFi firewall groups and rules
    • Port-based access control
    • MAC filtering if required
  • Guest portal requirements:
    • UniFi guest portal customization
    • Landing page branding
    • Authentication method (voucher, password, social)
    • Terms of service acceptance
    • Usage time limits
17. Plan Redundancy and Failover
SonicWall MikroTik UniFi
  • Dual WAN failover (SonicWall/MikroTik)
  • Switch stacking (UniFi switch redundancy)
  • Power redundancy (dual PSU, UPS)
  • UniFi controller backup
18. Develop Naming Conventions for Devices
  • Location-Type-Number format
  • Example: HQ-SW-01, HQ-AP-02
  • Consistent across all platforms
  • Document in spreadsheet

Pre-Configuration
19. Create Configuration Templates
SonicWall MikroTik UniFi
  • SonicWall base configuration:
    • Zone configuration (WAN, LAN, DMZ)

20. Configure Devices in Lab/Staging Environment
SonicWall MikroTik UniFi
  • Set management IPs:
    • Assign static IPs to all devices
    • SonicWall management IP (LAN interface)
    • MikroTik management IP
    • UniFi gateway, switches, APs
    • Document in IP spreadsheet
  • Configure VLANs and trunking:
    • Configure VLANs on UniFi gateway
    • Set trunk ports on UniFi switches
    • Configure MikroTik VLAN interfaces if used
    • Test inter-VLAN routing
  • Apply security policies:
    • Apply SonicWall security rules
    • Configure MikroTik firewall filters
    • Set UniFi firewall groups and rules
    • Enable IPS on SonicWall
    • Test rule effectiveness
  • Set up wireless controller and SSIDs:
    • Create WiFi networks in controller
    • Configure WPA2/3 Enterprise (RADIUS) if needed
    • Set channel width and power levels
    • Enable band steering and fast roaming
    • Configure guest portal
  • Configure monitoring/management access:
    • Configure SNMP on all devices
    • Set up syslog forwarding
    • Enable remote management (secure)
    • Configure email alerts
    • Test access from management workstation
21. Test Configurations
SonicWall MikroTik UniFi
  • Verify connectivity:
    • Test internet connectivity through SonicWall/MikroTik
    • Verify DHCP on all VLANs
    • Test DNS resolution
    • Confirm gateway redundancy
  • Test VLAN routing:
    • Test communication between allowed VLANs
    • Verify blocked traffic between isolated VLANs
    • Test guest network isolation
    • Confirm IoT VLAN restrictions
  • Validate security rules:
    • Test SonicWall IPS/GAV/Content Filtering
    • Verify firewall rule order and effectiveness
    • Test VPN connectivity
    • Check application control rules
  • Check wireless functionality:
    • Connect devices to all SSIDs
    • Test UniFi guest portal
    • Verify roaming between APs
    • Check band steering
    • Test WiFi speeds
22. Document All Configurations
SonicWall MikroTik UniFi
  • Export SonicWall configuration
  • Save MikroTik .rsc backup
  • Backup UniFi site
  • Document all settings in spreadsheet
  • Create network diagram with IPs
23. Create Device Labels with IP Addresses

  • Cable labels for identification
  • QR codes for quick access (optional)
